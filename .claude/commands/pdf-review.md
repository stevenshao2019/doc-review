# /pdf-review — PDF学术专著审校命令

## 调用方式

```
/pdf-review <pdf文件路径>
```

路径可以是：
- 绝对路径：`/Users/shaojun/.../*.pdf`
- 相对路径：`"./村庄与现代性...初排 - 2026.6.16.pdf"`
- 仅文件名：将在当前目录查找

## 功能

对PDF学术专著进行专业级审校（复用已验证的"动荡年代的私人历史"审校流程）。

发现文字错误、表达问题、事实风险、引文问题、译名问题，生成完整审校报告并输出人工确认包。

## 一、环境要求检查

执行PDF解析前，先检查以下工具是否可用：

| 工具 | 用途 | 检查命令 |
|------|------|---------|
| Python 3.8+ | 主处理语言 | `python3 --version` |
| PyMuPDF (fitz) 1.27+ | PDF解析与文本提取 | `python3 -c "import fitz; print(fitz.version)"` |
| Pillow | 图像处理 | `python3 -c "from PIL import Image; print('ok')"` |
| Tesseract + chi_sim + eng | OCR引擎 | `tesseract --list-langs` |

如缺少依赖，先安装再继续。

## 二、执行流程（7步）

### Step 1 — PDF解析

目标：提取PDF完整信息并进行结构分析。

必须输出到 `output/PDF审校/<PDF文件名>/analysis/` 目录。

**1.1 环境分析**

使用 Python + PyMuPDF 分析PDF属性：

```python
import fitz
doc = fitz.open(pdf_path)
# 提取：页数、PDF版本、文件大小、字体信息、版权信息、章节结构
```

输出文件：`环境分析报告.md`

报告格式必须与"动荡年代的私人历史"一致：

```markdown
# PDF审校环境分析报告

## 基本信息

| 项目 | 值 |
|------|-----|
| 文件名 | ... |
| 路径 | ... |
| 文件大小 | ... |
| 总页数 | ... |
| PDF版本 | ... |

## 1. 环境工具检查

| 工具 | 版本 | 状态 | 用途 |
|------|------|------|------|

## 2. PDF结构分析

### 2.1 内容结构

| 章节 | PDF页码 | 印刷页码(标) | 页数 |

### 2.2 版权信息

### 2.3 字体信息

## 3. 文字提取能力分析

### 3.1 提取数据概览

### 3.2 提取质量评估
```

**1.2 文本提取**

提取全部文本并保存：

```python
# 每页提取
for i, page in enumerate(doc):
    text = page.get_text("text", sort=True)
    pages_text.append(text)

# 全文合并
full_text = "\n==PAGE_BREAK==\n".join(pages_text)
```

输出文件：
- `analysis/全文文本.txt` — 全文纯文本，每页用 `==PAGE_BREAK==` 分隔
- `analysis/分页文本.json` — `[{page: N, text: "..."}]`
- `analysis/文本分块.json` — 按合理长度分块

**1.3 脚注提取**

使用正则提取脚注：

```python
# 常见脚注模式
footnote_patterns = [
    r'^[\s]*\*{1,3}\s*\d+\s*',     # *1 或 **1
    r'^[\s]*\d+\.[\s（(（(]',        # 1. 或 1（
    r'^[\s]*①',                      # 中文圈码
]
```

输出文件：
- `analysis/脚注提取.json`
- `analysis/脚注行.txt`
- `analysis/reconstructed_footnotes.json`

**1.4 分块准备**

将文本切分为 agent-level 处理块（每块约30-50页）：
- `analysis/文本分块.json` — `[{block_id, pages, text, char_count}]`

**1.5 初步词汇表提取**

- `analysis/英文作者名集.txt` — 提取文中出现的英文人名
- `analysis/book_titles.txt` — 提取书名/作品名
- `analysis/publishers.txt` — 提取出版社名

### Step 2 — 多Agent审校

复用"动荡年代的私人历史"的15 Agent体系。

**Agent分工维度：**

| Agent | 任务 | 审查内容 |
|-------|------|---------|
| 审校-A1~A4 | 文字审校（分块并行） | 错别字、漏字、语法、标点、表达 |
| 审校-B1 | 一致性检查 | 人名/术语前后一致性 |
| 审校-C1~C2 | 引文核查 | 引文格式、作者名、出版年、期刊名、页码 |
| 审校-D1 | 译名规范性核查 | 人名/书名翻译是否符合大陆学界规范 |
| 审校-E1 | 格式规范 | 中英文空格、标点全半角、连字符 |
| 审校-F1 | 学术细节核查 | 事实性错误、年代数据、专业术语 |
| 审校-G1~G3 | 交叉验证 | 对前序发现的争议性条目进行二次确认 |

**每条发现的JSON结构：**

```json
{
  "page": "页码（使用PDF页码）",
  "original": "原文内容",
  "suggestion": "修改建议",
  "reason": "修改原因（附依据）",
  "type": "A | B | C | D",
  "category": "typo | punctuation | format | grammar | expression | citation | name | academic"
}
```

**type分类定义：**

- **A类（确定错误）** — 错字、漏字、拼写错误、标点错误、格式错误、引文高危及确定错误、译名错误
- **B类（建议优化）** — 表达优化、建议核实引文、语言润色
- **C类（保持原文）** — 作者风格、修辞手法、无风险表达（当前流程中暂不产生）
- **D类（待确认）** — PDF文本提取导致的格式瑕疵（需人工对比原始PDF确认）

**category分类定义：**

| category | 说明 | 典型问题 |
|----------|------|---------|
| typo | 错字/漏字/拼写 | 错别字、缺字、英文拼写错误 |
| punctuation | 标点符号错误 | 半角冒号、引号使用、括号空格 |
| format | 格式规范问题 | 中英文空格、连字符空格、字体格式 |
| grammar | 语法结构问题 | 病句、搭配不当、因果连词缺失 |
| expression | 语言表达优化 | 冗余、重复、逻辑不清 |
| citation | 引文高危错误 | 作者名拼写错、出版年错、期刊名错、页码错 |
| name | 译名错误 | 人名/书名不符合大陆学界规范、前后不一致 |
| academic | 学术事实 | 专业术语、年代数据、事实性疑问 |

### Step 6 — 等待人工确认（Web 8230）

**6.1 Markdown确认系统接入**

当人工确认包生成后，将确认包内容推送到 Web 确认系统（端口8230）。

推送方式：通过 HTTP 将 Markdown 文件内容发送到 `http://localhost:8230/api/confirm`

推送内容结构：

```json
{
  "project": "<PDF文件名>",
  "output_dir": "output/PDF审校/<PDF文件名>/",
  "b_confirm": "2-B类问题人工确认.md 的完整内容",
  "d_confirm": "3-D类学术问题确认.md 的完整内容",
  "a_pending": "4-A类未完成问题.md 的完整内容",
  "status_url": "http://localhost:8230/api/status/<project>"
}
```

**6.2 等待确认完成**

- 轮询确认状态：`GET http://localhost:8230/api/status/<project>`
- 等待状态变为 `confirmed` 或超时
- 确认结果包含：
  - B类：每条 `radio` 选择结果（建议修改 / 保持原文）
  - D类：每条 `checkbox` 确认结果

**6.3 回写确认结果**

将 Web 系统的确认结果写回人工确认包：

- `2-B类问题人工确认.md` — 追加确认标记
- `3-D类学术问题确认.md` — 追加确认标记

## 三、输出规范

### 3.1 异常处理

| 情况 | 处理 |
|------|------|
| PDF无法打开 | 报错退出，提示检查文件 |
| 文本提取为空 | 提示尝试OCR路径 |
| OCR也未提取到 | 提示PDF可能是扫描件，需人工处理 |
| 无发现问题 | 输出空报告并备注"未发现明显问题" |
| 确认系统不可达 | 降级为本地确认模式（直接保存确认包到本地） |

### 3.2 多PDF并行处理

当检测到输入为PDF列表时：

1. 为每个PDF创建独立子目录：`output/PDF审校/<PDF文件名1>/`、`output/PDF审校/<PDF文件名2>/`
2. 每个PDF独立执行完整7步流程
3. 输出目录相互隔离，不共享状态
4. 最终生成并行审校总览：`output/PDF审校/并行审校总览.md`

### 3.3 本地确认模式（降级方案）

当 Web 8230 确认系统不可用时，输出确认包到 `人工确认包/` 目录，提示用户：
